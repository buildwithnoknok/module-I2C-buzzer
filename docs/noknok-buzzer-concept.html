Musician — Buzzer

CH32V003 · 16KB flash
What the firmware does with each command
0x00
→
Stop PWM immediately. Clear busy flag.
0x01
→
Read freq (2 bytes), duration, volume. Calculate PWM prescaler. Set duty cycle. Start timer. Return to I²C immediately — timer runs in background.
0x02
→
Read tune ID. Look up note sequence in flash. Play each note in sequence, then silence. Return to I²C immediately — tune runs in background.
READ
→
Reply with 1 byte: 0x01 = playing, 0x00 = idle.

Why fire and forget?

The Pico sends a command and moves on instantly. The CH32V003 handles all timing internally using its hardware timer. The Pico never has to wait or count milliseconds — it's free to do other things while the buzzer plays.

Why tunes on the CH32V003?

Tunes are always available, no matter what app is on the Pico. The Pico sends 2 bytes. The CH32V003 knows the rest. Adding a new tune means updating the buzzer firmware — independent of any Pico app.

Memory usage

CH32V003 has 16 KB flash. A tune with 25 notes costs ~100 bytes. Ten tunes = ~1 KB. Firmware logic = ~3–4 KB. Plenty of room for the standard register header and OTA bootloader too.
